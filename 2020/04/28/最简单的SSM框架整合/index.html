最简单的SSM框架整合
发表于 2020-04-28| 更新于 2020-04-28|SSM学习
字数总计:2.5k|阅读时长: 11 分钟
| 阅读量:

网上的SSM整合的文章都太复杂了，非常不适合像我这样的小白，而且SSM素有“配置地狱”的说法，所以我想写一篇仅仅保留最基本的配置的SSM框架整合文章。更高级的功能（如事务管理、拦截器等等）可以根据业务或学习的需要以后自己配置。

@TOC

项目环境

  • 项目管理工具：maven
  • 数据库：mysql 5.7.29
  • Web 应用服务器：tomcat 8.5
  • java版本：java8
    上面的东西可以根据自己的需求改变

写在前面

我推荐大家先从底层往上配置，即mybatis->spring->spring mvc的顺序配置，并且配好一个模块就测试一个模块，这样不容易混乱，即使报错也容易修复。

==注意：在创建maven项目的时候一定要选择war包==

tips:有些时候eclipse通过maven创建web项目可能没有web.xml文件，可以通过以下方法解决

下面bean里的id有些是固定的，有些是自定义的，我推荐用用系统给的id（按alt+/补全）

mybatis-spring

导入jar包

既然要连接mybatis和spring，那么mybatis和spring的核心jar包一定要导入进去，还要在导入一个连接二者的jar包，还有mybatis连接数据库的那些jar包。

  • mybatis核心包
  • spring核心包
  • 连接mybatis和spring和jar包
  • 连接mybatis和数据库的jar包
    • 数据库连接池
    • 数据库驱动

tips：大家在maven仓库找spring核心包的时候可以只搜索spring web mvc，里面不仅包含spring的核心包，还有springmvc的包

DAO层

映射类

mybatis的特点就是像操作类一样操作数据库，我推荐先把映射类写好。
假设是一张关于商店商品的表

不要忘了toString方法

编写映射接口（.java）

mybatis能够自动继承接口并实例化，以后我们可以直接调用接口中的方法

编写映射文件（.xml）

之后就把mapper.xml文件写了。我推荐专门建一个mapper文件夹来管理映射文件。
mapper.xml可以由插件帮助生成，不过我推荐保留一份模板，用的时候复制上去，本身这个文件也不复杂
注意id值要和接口方法名一样

编写spring bean

spring和mybatis整合的bean主要由三块组成

  • BasicDataSource（数据库配置）
  • SqlSessionFactoryBean（产生sqlsession）
  • MapperScannerConfigurer（扫描mapper接口，自动生成bean）
    • bean的id值是首字母小写的类名

由于数据库的配置信息在文件里写死的话会很不好管理，所以一般都把它写在外面，有<context:property-placeholder/>标签引入。
我们一般把spring的配置文件称为applicationContext

编写数据库信息（db.properties）

当jdbc和mysql版本不兼容时要手动指定是否使用SSL连接

编写测试类（Service层）

到此，spring和mybatis的整合就做好了，我们只要写个测试类测试一下就好了，这个类的代码和Service层的逻辑是一样的，可以直接用。

一般Service文件会先建一个接口，然后创建它的实现类（ServiceImpl）。

springmvc

后端已经搭建完成，现在开始搭建前端，前端的配置比后端要简单一些

首页（index.jsp）

因为仅用来测试，所以只在首页写一个超链接发送请求。

Controller层

Controller层可以对springmvc拦截的请求进行处理

最后程序会请求转发success加上前后缀（下一节会配置前后缀）的文件。

spring-mvc.xml

springmvc和配置文件主要包含三个东西

  • 注解扫描驱动
  • 注解扫描器
  • 视图解析器

编写web.xml

web.xml文件用来处理网页请求相关的操作，主要包含两块。

  • ContextLoaderListener（监听SpringIOC容器的启动）
  • springDispatcherServlet（拦截网页的请求）

alt+/自动生成，只要改改一些参数就好了，简单方便

请求转发的结果（/WEB-INF/view/success.jsp）

由于是测试，只需要随便写点东西就可以了。
我推荐把请求转发的结果放在WEB-INF文件夹并创建专门的文件夹管理，因为WEB-INF文件夹权限较高，直接请求是无法进入的。

总结

“配置地狱”果然名不虚传，这些配置去做真正的业务肯定是不够的，但是已经足够让SSM跑起来了，大家可以根据自己的实际情况添加其他配置。
